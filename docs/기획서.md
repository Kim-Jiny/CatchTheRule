# CatchTheRule — 규칙찾기 게임 기획서

> 작성일: 2026-06-04 · 상태: 초안(v0.1)

---

## 1. 한 줄 소개

수열·패턴의 **규칙을 찾아 다음(혹은 빈칸)을 맞히는** 싱글플레이 퍼즐 게임.
단계가 올라갈수록 어려워지고, 막히면 힌트를 쓸 수 있다. 타임어택으로 랭킹에 도전한다.

## 2. 확정된 결정사항 (Q&A 반영)

| 항목 | 결정 | 메모 |
|---|---|---|
| 플랫폼 | **iOS(SwiftUI) 우선 + Android(Compose) 병행** | `iOS/`, `Android/` 디렉토리 분리. 동일 `puzzles.json` 스키마 공유 (구조는 [README](../README.md) 참고) |
| 백엔드 | **본인 Minigame 서버** | 닉네임+점수 랭킹 저장. 앱은 연동 레이어만 두고 실제 API는 추후 결합(§8) |
| 수익화 | **광고 없는 MVP** | 힌트는 무료 또는 일일 제한. 추후 AdMob 리워드 광고 자리만 확보 |
| 로그인 | 없음 | 진행도는 로컬 저장. 랭킹은 닉네임만 입력 |

## 3. 타깃 & 플랫폼

- **타깃**: 두뇌 퍼즐을 가볍게 즐기는 라이트 유저 (전 연령)

- 세로 모드 전용, 다크/라이트 대응(다크 우선 톤)

## 4. 게임 디자인

### 4.1 핵심 루프
```
홈에서 "이어하기" → 퍼즐 제시 → 규칙 추론 → 정답 입력
→ 정답: 별/진행도 획득, 다음 단계  |  오답: 재시도 또는 힌트
```

### 4.2 퍼즐 유형 (난이도 태그 포함)

규칙은 **타입 코드**로 관리해 콘텐츠를 쉽게 늘린다. 시작 라인업:

| 코드 | 규칙 | 예시 | 난이도 |
|---|---|---|---|
| `arith` | 등차수열 | 1 2 3 4 **?** → 5 | ★ |
| `odd_even` | 홀/짝 | 1 3 **?** 7 → 5 | ★ |
| `step` | 일정 간격 | 2 4 6 8 **?** → 10 | ★ |
| `geo` | 등비수열 | 2 4 8 16 **?** → 32 | ★★ |
| `square` | 제곱수 | 1 4 9 16 **?** → 25 | ★★ |
| `fib` | 피보나치 | 1 1 2 3 5 **?** → 8 | ★★ |
| `diff_inc` | 차이가 늘어남 | 1 2 4 7 11 **?** → 16 | ★★★ |
| `prime` | 소수 | 2 3 5 7 **?** → 11 | ★★★ |
| `alt` | 두 규칙 교차 | 1 10 2 20 3 **?** → 30 | ★★★ |
| `letter` | 알파벳 패턴 | A C E G **?** → I | ★★ |
| `shape` | 도형/색/회전(시각) | (도형 회전) | ★★★ |

> `?`는 끝이 아니라 **중간**에도 올 수 있음(예: 1 3 **?** 7). 빈칸 위치는 퍼즐 데이터에 포함.

### 4.3 난이도 곡선

- **챕터(World) 단위**로 묶음. 각 챕터 10~15퍼즐.
  - Ch.1 기초(arith/odd_even/step) → Ch.2 곱셈·제곱 → Ch.3 피보나치·차이증가 → Ch.4 소수·교차 → Ch.5 혼합/시각
- 챕터 내에서도 숫자가 커지거나 빈칸이 앞쪽으로 이동하며 점진 상승.
- **별점 3개** 보상: 무힌트 + 빠른 시간 = 별 3개 (재도전 동기).

### 4.4 정답 입력 방식

- 기본: **숫자 키패드**(0–9, 지우기, 제출) — 직접 입력이 "규칙을 안다"는 성취감↑
- 일부 유형(letter/shape): **4지선다** 보기
- 퍼즐 데이터의 `inputType` 필드로 분기

### 4.5 힌트 시스템

3단계 점진 공개:
1. **규칙 카테고리** 귀띔 ("차이를 보세요")
2. **부분 공개** (인접 항의 관계 1개)
3. **정답 근접** (마지막 결정적 단서)

- MVP: 힌트는 일일 N개 무료 또는 무제한(설정 가능 상수).
- 추후: 리워드 광고 1회 = 힌트 1개 (자리만 확보, §10).

## 5. 화면 구조 (탭 네비게이션)

3개 탭 (`TabView`):

### 🏠 홈 탭
- 현재 진행: "Chapter 2 · Stage 14 도전 중"
- **이어하기** 큰 버튼(Primary CTA)
- 챕터별 진행 바 / 획득 별 총합
- (선택) 오늘의 퍼즐

### 🏆 도전 탭 (타임어택 + 랭킹)
- **타임어택**: 60초 동안 최대한 많이 풀기 → 점수 = 정답 수(+속도 보너스)
- 종료 후: 닉네임 입력 → Minigame 서버 제출
- **랭킹 보드**: 전체/주간, 내 순위 하이라이트
- 닉네임은 로컬 저장 후 재사용

### ⚙️ 설정 탭
- 문의하기(메일 `mailto:` 또는 폼)
- 사운드/햅틱 토글, 다크모드
- 닉네임 변경
- 진행도 초기화
- 약관/개인정보처리방침, 버전 정보
- (추후) 광고 제거 IAP 복원

## 6. 데이터 모델 & 콘텐츠 포맷

### 6.1 퍼즐 JSON (플랫폼 비종속 — iOS/안드 공용)

```json
{
  "id": "ch1_s03",
  "type": "odd_even",
  "chapter": 1,
  "order": 3,
  "sequence": [1, 3, null, 7],
  "answer": 5,
  "inputType": "keypad",
  "choices": null,
  "hints": [
    "숫자 사이의 간격을 보세요.",
    "1과 3의 차이는 2예요.",
    "홀수가 순서대로 나열돼요."
  ],
  "explanation": "연속된 홀수: 1, 3, 5, 7"
}
```

- 시각 퍼즐(`shape`)은 `sequence` 대신 `assets`/`render` 필드로 확장.
- 전체 콘텐츠는 `puzzles.json`(챕터별 분할 가능)으로 번들 동봉 → **서버 불필요**.

### 6.2 진행 상태 (로컬)

```json
{
  "currentChapter": 2,
  "currentStage": 14,
  "stars": { "ch1_s01": 3, "ch1_s02": 2 },
  "hintsRemaining": 5,
  "nickname": "고수123",
  "settings": { "sound": true, "haptics": true }
}
```

저장: iOS `UserDefaults`(+Codable) 또는 파일. 추후 동기화 불필요.

## 7. 기술 스택 & 아키텍처 (iOS 먼저)

- **SwiftUI + Swift 5.9 / iOS 16+**
- 패턴: **MVVM**, `@Observable`(또는 `ObservableObject`)
- 레이어 분리(안드 포팅 대비):
  - `Core/` — 퍼즐 모델, 규칙 검증, 진행 로직 (UI 비종속, 순수 Swift)
  - `Content/` — `puzzles.json` 로더
  - `Features/` — Home / Challenge / Settings 화면
  - `Services/` — `RankingService`(프로토콜) + `MinigameRankingService` 구현
  - `DesignSystem/` — 컬러·타이포·컴포넌트
- **랭킹은 프로토콜로 추상화** → 서버 스펙 확정 전엔 Mock 구현으로 개발 진행 가능.

```swift
protocol RankingService {
    func submit(score: Int, nickname: String, mode: GameMode) async throws
    func leaderboard(mode: GameMode, scope: Scope) async throws -> [RankEntry]
}
```

## 9. 디자인 시스템 (세련된 톤)

- **무드**: 미니멀 + 다크 베이스 + 포인트 그라데이션. 큰 숫자 타이포가 주인공.
- **컬러(초안)**
  - 배경: `#0E1116` ~ `#151A22` (딥 네이비/차콜)
  - 포인트: 바이올렛→시안 그라데이션 (`#7C5CFF` → `#39E5C8`)
  - 정답: 민트 그린 / 오답: 소프트 레드
- **타이포**: 숫자는 라운드 계열(SF Rounded), 본문 SF Pro. 수열 항은 카드/캡슐로.
- **모션**: 정답 시 스케일 팝 + 햅틱, 오답 시 셰이크. 단계 전환 부드러운 페이드/슬라이드.
- **접근성**: 동적 타입, 색맹 대비(정/오답에 아이콘 병행).

> 원하시면 이 톤으로 **Figma 시안**(홈/퍼즐/타임어택/랭킹/설정 5화면)을 먼저 만들어볼 수 있어요.
